# Contributing

Thanks for considering a contribution to this Etymolt repository. This document is the short version; the full org-wide conventions live in [`AGENTS.md`](./AGENTS.md).

## Before you contribute

- **For protocol/normative changes**, open an issue first. PRs that land normative changes without a precursor issue will be closed and re-opened against the issue thread.
- **For code/docs changes**, run the local tests and lints before opening a PR. Each repo's CI workflow is the source of truth for what must pass.
- **For security issues**, do NOT file a public issue — see [`SECURITY.md`](./SECURITY.md).

## Voice + style

Etymolt's voice is reportorial, not advisory. We are honest about coverage. Every numeric claim cites a source.

- **Don't use:** "robust", "comprehensive", "leverage", "AI-powered", "killer", "dominate".
- **Do use:** verdict (not result), axes (not categories), clearance signal (not legal advice), records of record (not sources), signed result (not "secure").
- **End in an action**, not a vibe.

See [`AGENTS.md`](./AGENTS.md) for the full voice lock.
