# AGENTS.md

This file is for AI coding agents (Claude Code, GitHub Copilot, Cursor, Aider, Codex, Continue) and the operators running them. It describes how AI agents should contribute to this repository.

## Project context

This repository is part of **Etymolt** — the fact-check layer for LLM-generated names. The org-wide overview lives at [github.com/etymolt](https://github.com/etymolt). The Bureau Model framing (we report on records of record; we do not opine on infringement; we are not a law firm) applies to every artifact this org ships.

## Voice locks

The following phrasings are locked across every Etymolt repo and must not drift:

- **Tagline (verbatim):** "The fact-check layer for LLM-generated names."
- **Closer (verbatim):** "Naming, attested."
- **The three load-bearing concepts:** Bureau Model · EVP/1 (Etymolt Verdict Protocol) · the five axes (trademark, domain, cultural, sound, pronunciation).
- **Verdict labels:** PROCEED / ITERATE / DECIDE / ABANDON / INSUFFICIENT_SIGNAL.

**Etymolt SAYS / does NOT say:**
- "verdict" not "result"; "axes" not "categories"; "clearance signal" not "legal advice"; "records of record" not "sources"; "signed result" not "secure"; "fact-check layer" not "verification layer".

**Etymolt is NOT:** a name generator, a law firm, a recommendation engine, a SaaS toy. Every public surface should be clear about what we are not.

## Repository conventions

- **License:** every public Etymolt repo carries a clear license. Spec/protocol/docs repos: CC-BY-4.0. Code/SDK/CLI repos: Apache-2.0 unless stated otherwise. Trademarks (the Etymolt name + marks) are not licensed under either.
- **Commit messages:** present tense, imperative ("Add X" not "Added X"). Conventional Commits style (`feat:`, `fix:`, `docs:`, etc.) preferred. Do NOT include AI-agent identifying details (model name, agent ID, "Generated by Claude") in commit messages — keep them invisible.
- **Branch naming:** `feature/<short-name>`, `fix/<short-name>`, `docs/<short-name>`.

## What AI agents should do

- **Run tests + lint before any PR.** Each repo's CI workflow is the source of truth for what must pass.
- **Anchor issues to a specific file/line.** No "this is broken somewhere" issues.
- **Render the disclaimer verbatim.** If your output surfaces an EVP/1 verdict to a user, the `disclaimer` field MUST be rendered verbatim. This is the Bureau Model anchor.
- **Respect temporal validity.** If your output surfaces a verdict more than 24 hours after `issued_at`, render a staleness banner. See [VERDICT-VALIDITY.md](https://github.com/etymolt/evp-spec/blob/main/docs/VERDICT-VALIDITY.md).
- **Use `hello@etymolt.com` for any contact reference** (not personal addresses, not legacy aliases).

## What AI agents should NOT do

- Do not invent verdicts. EVP/1 verdicts are issued by conformant issuers; an AI assistant generating a verdict locally is not an EVP/1 issuer.
- Do not strip the `disclaimer` field from any rendered verdict. It is signed; modifying it invalidates the signature.
- Do not modify `issued_at`, `valid_until`, or the signature fields on re-rendering.
- Do not add marketing adjectives ("robust", "comprehensive", "leverage", "AI-powered", "killer", "dominate") to any user-facing copy.

## When in doubt

Open an issue with a clear section/file reference. Specificity is how we triage at scale.
